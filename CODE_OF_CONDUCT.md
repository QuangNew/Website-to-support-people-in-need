# Code of Conduct

## Our commitment

We want ReliefConnect to be a welcoming, respectful, and constructive project for everyone who contributes, reports issues, reviews code, or uses the repository.

Participants are expected to:
- Be respectful, patient, and professional.
- Assume good intent while giving clear, actionable feedback.
- Focus discussions on improving the project.
- Respect different backgrounds, skill levels, and viewpoints.

## Expected behavior

Examples of positive behavior include:
- Using inclusive and respectful language.
- Accepting feedback gracefully.
- Explaining technical decisions without personal attacks.
- Helping others learn and contribute safely.
- Reporting security concerns privately instead of exposing users to unnecessary risk.

## Unacceptable behavior

Examples of unacceptable behavior include:
- Harassment, hate speech, or discriminatory language.
- Personal attacks, insults, or trolling.
- Publishing private information without permission.
- Spamming, derailing discussions, or creating a hostile environment.
- Sharing exploit details, credentials, or harmful instructions in public channels.

## Scope

This Code of Conduct applies in:
- GitHub issues
- Pull requests
- Code reviews
- Discussions related to this repository

## Reporting

If you experience or witness unacceptable behavior, report it privately to the project maintainer through GitHub rather than opening a public issue. Include links, screenshots, or other context that may help with review.

For security vulnerabilities, do not use public issues. Follow the process in [SECURITY.md](SECURITY.md).

## Enforcement

Project maintainers may remove comments, close discussions, reject contributions, or block contributors whose behavior violates this Code of Conduct.

Actions will be taken based on the severity and pattern of the behavior, with the goal of protecting the community and keeping collaboration productive.
